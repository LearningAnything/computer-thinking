# 动态规划

## 求解步骤

动态规划是一种通过把原问题分解为相对简单的子问题的方式来求解复杂问题的方法。

对动态规划问题求解的一般步骤:

1. 定义子问题 把原问题分解为足够简单的子问题。
2. 确认状态 确定哪些变量可以描述一个子问题的状态,这样就可以避免重复计算。
3. 确定状态转移方程 找到状态转移方程,表示子问题之间的关系,如何由一个子问题的解得到下一个子问题的解。
4. 确定边界条件 子问题的初始解,给状态转移方程一个起点。
5. 计算顺序 从上而下还是从下而上计算状态转移方程,存储子问题的解以备快速查找。
6. 优化存储 存储子问题的解时,可以只存储必要的数据,节省空间复杂度。

总结一下,动态规划通过把问题分解,明确状态和状态转移方程,边界条件,计算顺序,存储优化等步骤,使得复杂问题变成简单子问题的解的组合,从而高效求解。



## 方法

1. 多看动态规划的典型例题,比如背包问题、最长递增子序列等,对问题范式有深刻理解。
2. 学会分解问题,抓住状态和状态转移equation,把大问题分解为子问题。
3. 多思考如何定义状态,确定哪些变量可以描述问题的状态。状态设计好坏直接影响问题的难易程度。
4. 掌握状态转移方程的求解技巧,利用数学归纳法、构造最优解等方法求解方程。
5. 学会两种常见的计算顺序:自底向上和自顶向下,了解它们的区别。
6. 学会存储子问题结果,利用备忘录或者DP table优化递归及避免重复计算。
7. 分析时间和空间复杂度,优化存储方式,只存储必要数据。
8. 在练习时,可以先人肉模拟小数据量的示例,辅助理解题意和setState转移过程。
9. 多做练习,开始从简单题开始,逐步进行面向复杂题,重复练习同类型题目。
10. 学习优秀代码,理解别人高效解题思路。记录常见解法模板,运用框架思维。

多练习,多总结,多观察优秀代码,掌握动态规划的题型和解题套路,才能更好地理解和运用动态规划。



## 泛式

动态规划常见的问题范式包括:

1. 矩阵路径问题 典型例题有计算从左上角到右下角的最短路径长度等。通过状态转移方程求解。
2. 整数划分问题 将一个整数划分成若干个整数的和,通常要求满足某些条件。比如装配背包的最大价值。
3. 最长公共子序列问题 求两个序列的最长公共子序列长度,典型的二维动态规划。
4. 最长递增子序列 求给定序列的最长递增子序列长度,一维动态规划。
5. 计数类问题 求解可行解的总数,例如制定总步数为n的方式数。
6. 最小编辑距离 transforming one string into another 需要的最少编辑操作次数。
7. 背包问题 有限容量的背包,如何选择物品放入背包达到目标(最大价值、最小重量等)。
8. 背包问题的变体 01背包、完全背包、多维费用背包、混合背包等。
9. 并查集与最小生成树问题 使用并查集判断图中是否存在环等。

以上是一些常见的动态规划问题范式,通过大量练习内化这些模式,可以事半功倍。



## 其他问题

1. 动态规划和递归、分治法、贪心法的不同之处。

动态规划和递归都把原问题分解为子问题求解,但动态规划会保存子问题的解,避免重复计算。分治法也分解为子问题,但子问题往往不相交。贪心法更倾向于做出局部最优选择。

2. 动态规划的重叠子问题性质。

重叠子问题是指递归算法中重复计算相同子问题的情况,动态规划利用备忘录消除重叠子问题。

3. 动态规划和备忘录、DP表的关系。

备忘录保存递归中已解决子问题的结果,避免重复计算。而DP表事先建表,表格中的每一格是一个子问题的解。

4. 如何判断一个问题可以不能用动态规划来解决。

具有最优子结构性质,可以分解为相互依赖的子问题,这时就可以考虑用动态规划来求解。

5. 动态规划的时间复杂度和空间复杂度分析。

动态规划 Algorithm 的时间复杂度受状态数和状态转移次数影响。空间复杂度主要取决于状态的存储需求。
